Why Was Databricks Created?
Before Databricks:
A company had to Create VMs, Install Spark, Configure Spark, Configure Hadoop, Configure Security, Configure Storage, Monitor Clusters, Handle Failures
This required lots of DevOps work. Now Databricks automates most of this.

What is databricks?
Databricks is a cloud-based data lakehouse platform built on Apache Spark that enables organizations to ingest, process, transform, analyze, and 
manage large-scale data workloads using distributed computing, Delta Lake, and collaborative workspaces for data engineering, analytics, AI, and machine learning.

Does Databricks store data?
Databricks is primarily a compute and analytics platform. The actual data is usually stored in cloud storage such as Azure Data Lake Storage, Amazon S3, or Google Cloud Storage. 
Databricks uses Spark to process the data and Delta Lake to manage it, while the storage remains in the underlying cloud storage layer.

Control Plane
Handles: Notebook UI, Saving notebook, User authentication, Cluster creation, Job scheduling
It does not process your data.

Data Plane
Contains: Driver Node, Worker Nodes, Executors
This is where Spark executes

Simple Example:
will write in notebook print("Hello") and save (This all will happen in Control Plane)
but once you click Run, the code is sent to the Driver in the Data Plane for execution.

Manage table and extermal table -
In Manage table metadata and actual data both managed by metastore
In External table only metadata is managed by metastore

Unity Catalog -
Scenario: In a modern data platform, data is stored across multiple sources such as:
Azure SQL Database
Cloud Storage
Azure Data Lake Storage Gen2 (ADLS Gen2)
Databricks Workspace

Problems Without Unity Catalog:
Difficult to manage data access permissions across different storage systems.
No centralized governance for data assets.
Hard to track who accessed, modified, or used the data.
Managing security and auditing separately for each storage system becomes complex.

Solution – Unity Catalog:
Unity Catalog provides a centralized governance layer for all data assets. It helps:
Manage permissions and access control from a single place.
Track data usage through auditing and monitoring.
Maintain data security and compliance.
Organize data using Catalogs, Schemas, and Tables.
Govern data across Databricks workspaces and multiple storage locations.

In Short:
Unity Catalog is Databricks' centralized data governance solution that provides unified access control, auditing, lineage tracking, and security for all data assets across the data platform.
